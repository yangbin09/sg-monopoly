# 三国大富翁游戏 测试文档

## 项目概述

- **项目名称**: 三国大富翁游戏
- **技术栈**: HTML + CSS + JavaScript (纯前端)
- **测试方式**: 浏览器自动化 + 手动测试

---

## 功能测试要点

### 1. 角色选择系统

| 测试点 | 预期结果 | 测试方法 |
|--------|----------|----------|
| 初始界面显示 | 显示"选择你的英雄"标题，4张角色卡片 | 页面加载验证 |
| 角色数量 | 4个角色(刘备/关羽/张飞/诸葛亮) | DOM验证 |
| 角色信息 | 显示角色名、技能描述、头像 | UI验证 |
| 选择流程 | 选2个角色后进入游戏 | 点击2张卡片 |
| 重复选择拦截 | 已选角色再次点击提示"已被选择" | 重复点击测试 |
| 选择顺序 | 先选刘备，再选关羽，记分板顺序正确 | 记分板验证 |

### 2. 棋盘系统

| 测试点 | 预期结果 | 测试方法 |
|--------|----------|----------|
| 棋盘尺寸 | 5x5=25格子，外圈16格可用 | DOM计数 |
| 格子类型 | start/property/event/store/tax 五种 | CSS类验证 |
| 起点位置 | 左下角，绿色背景 | 样式验证 |
| 地产格子 | 显示名称+价格 | 内容验证 |
| 事件格子 | 显示名称+类型 | 内容验证 |
| 棋子显示 | 玩家头像显示在当前位置 | UI验证 |

### 3. 游戏流程

| 测试点 | 预期结果 | 测试方法 |
|--------|----------|----------|
| 初始金币 | 每玩家1000金币 | 记分板验证 |
| 掷骰子按钮 | 橙色按钮，可点击 | 点击测试 |
| 骰子范围 | 1-6随机数 | 多次点击验证 |
| 棋子移动 | 按骰子点数前进 | 位置前后对比 |
| 经过起点 | 获得200金币奖励 | 金币变化验证 |
| 消息显示 | 每次行动显示提示 | 消息区域验证 |

### 4. 角色技能

| 角色 | 技能 | 测试点 |
|------|------|--------|
| 刘备 | 购买地产时+50金币 | 购买后金币验证 |
| 关羽 | 支付租金时-10金币 | 付租时金额验证 |
| 张飞 | 点数<3时自动+1 | 小点数掷骰验证 |
| 诸葛亮 | 负面事件免疫 | 触发负面事件验证 |

### 5. 地产系统

| 测试点 | 预期结果 | 测试方法 |
|--------|----------|----------|
| 购买确认 | 弹窗确认是否购买 | 弹窗测试 |
| 资金不足 | 无法购买，提示资金不足 | 资金少时测试 |
| 租金支付 | 停在别人地产上需付租 | 对战测试 |
| 地产回收 | 破产后地产归零 | 破产测试 |

### 6. 事件系统

| 事件ID | 格子名 | 效果 |
|--------|--------|------|
| yellowTurban | 黄巾起义 | -100金币 |
| peachOath | 桃园三结义 | +150金币 |
| redCliffs | 赤壁之战 | +200金币 |
| emperor | 朝见皇帝 | +100金币 |
| ambush | 埋伏 | -50金币 |
| strategist | 谋士策略 | +80金币 |
| store | 商店 | 提示未营业 |

### 7. 游戏结束

| 测试点 | 预期结果 |
|--------|----------|
| 单人存活 | 存活者胜利 |
| 全部破产 | 平局 |
| 按钮禁用 | 游戏结束后按钮不可点击 |

---

## UI/UX 测试要点

1. **响应式布局** - 不同窗口大小下棋盘是否正常显示
2. **悬停效果** - 角色卡片悬停上浮
3. **当前玩家高亮** - 橙色边框标识
4. **消息滚动** - 消息区域自动滚动到底部
5. **禁用状态** - 按钮禁用时半透明
